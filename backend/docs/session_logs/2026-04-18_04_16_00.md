## What Was Implemented Or Changed

- Added provision engine package under `src/provisions/`.
- Implemented `src/provisions/ppna.py` with prorata temporis PPNA calculation from canonical preprocessing rows, user-provided closing date support, per-row audit output, and per-network/product aggregation.
- Implemented `src/provisions/pe.py` with the settled PE formula:
  - `PE = 0` when `RT_N <= 0`
  - otherwise `PE = min(rate_rt * RT_N, rate_avg * moyenne(A1, A2, A3))`
- Loaded PE legislative coefficients from `src/config/legislative.yaml` instead of hardcoding them.
- Implemented `src/provisions/sap.py` using the workbook-driven aggregation logic:
  - include declared amount only when the closing date is on/after occurrence date and on/before notification date
  - blank notification dates currently produce zero SAP, matching the workbook formula reality
- Added provision exports in `src/provisions/__init__.py`.
- Added unit tests for PPNA date logic and leap-year handling, PE branch behavior and configurable rates, and SAP date-window aggregation behavior.
- Added integration/regression tests for PPNA, PE, and SAP on the sample workbooks.
- Adjusted PPNA behavior for zero-duration policies so they are retained in audit output with zero PPNA instead of aborting the run.
